def print_items(n):
    for i in range(n):
        for j in range(n):
            print(i,j)

print_items(10)

Simple Explanation

Imagine you have a list of nnn items. If you have to compare each item with every other item, you end up doing n×nn \times nn×n comparisons. This is like having a grid of nnn rows and nnn columns, and you need to look at each cell in the grid.

Real-life Analogy

Think of a classroom where every student needs to shake hands with every other student. If there are nnn students, each student needs to shake hands with nnn others. So, the total number of handshakes will be about n×nn \times nn×n.

Visual Representation

Imagine a square grid:

  • Each side of the grid has n cells.

    nn

  • To cover the entire grid, you need to visit n×n cells.

    n×nn \times n

Key Indicator

Whenever you see a situation with two nested loops, each running nnn times, you are likely dealing with O(n2)O(n^2)O(n2) complexity.

Summary

O(n2)O(n^2)O(n2) means: The time it takes to complete the task grows quadratically as the number of items nnn grows. If you double the number of items, the time to complete the task roughly quadruples.

By remembering these simple points and analogies, you can easily recognize and understand O(n2)O(n^2)O(n2) complexity.

O(1): Constant time as n increases number of operations will be the same

O(1), also known as constant time complexity, refers to an algorithm or operation that takes a constant amount of time to execute, regardless of the size of the input data.

Key Characteristics

  1. Independent of Input Size: The execution time does not change with the size of the input. Whether the input size is 1, 10, or 1,000,000, the operation will take the same amount of time.
  1. Fixed Number of Operations: The algorithm performs a fixed number of step

O(log n)

O(log n), or logarithmic time complexity, refers to an algorithm where the time it takes to complete its task increases logarithmically as the size of the input data increases. This means that the time increases very slowly relative to the increase in the size of the input.

Key Characteristics

  1. Logarithmic Growth: As the input size n increases, the time taken grows logarithmically. This is typically base 2, but can be any base.

    nn

  1. Halving: The algorithm often involves repeatedly dividing the input size by a constant factor (usually 2) at each step, significantly reducing the problem size quickly.
